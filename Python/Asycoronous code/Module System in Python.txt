1. What is a Module?

Answer:
A module is a Python file (.py) containing reusable code such as functions, classes, and variables.

2. Why Do We Use Modules?

Answer:
To organize code, improve reusability, reduce duplication, and make applications easier to maintain.

3. What are the Types of Modules?

Answer:

Built-in Modules (math, os, sys)
User-Defined Modules (calculator.py)
Third-Party Modules (numpy, pandas)
4. What is a Package?

Answer:
A package is a directory containing multiple related modules.

Example:

ecommerce/
├── __init__.py
├── user.py
├── payment.py
5. What is __init__.py?

Answer:
A special file that runs when a package is imported and is used to initialize the package and manage imports.

Answer:

Create module object
Compile to bytecode
Execute module code
Store in cache
Bind module name
7. What is sys.path?

Answer:
A list of directories where Python searches for modules.

8. What is sys.modules?

Answer:
A cache that stores all loaded modules so they are imported only once.

9. Why is a Module Imported Only Once?

Answer:
Because Python stores loaded modules in sys.modules and reuses them.

10. What is __name__?

Answer:
A special variable that stores the name of the current module.

14. What is __pycache__?

Answer:
A directory where Python stores compiled bytecode (.pyc) files.
